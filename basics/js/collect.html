    面试题汇总

    4/24/2021
    • http协议
      • 超文本传输协议，是互联网上应用最为广泛的一种协议；
      • http报文有起始行( start line ) 头部( header )和 主题 ( body ) 三部分组成，起始行是对报文进行的描述，头部包含报文的一些属性 ，主体包含报文的数据；
      • 报文分为请求报文( request message ) 和响应报文( response message )
      • http1.0和http1.1区别
        1. 节约宽带: 1.1版本支持只发送header信息(不携带任何body信息)，如果服务端认客户端有权限请求服务器，则返回100，客户端收到100后才开始把请求body发送到服务器，如果返回401，客户端就不用发送请求了
        2. 长连接： 1.1版本支持长连接和请求的流水线处理，在一个tcp连接通道中，可以进行多个http请求和相应，减少建立和关闭通道的消耗和延迟，在http1.1中默认开启长连接(keep-alive),一定程度上弥补了http1.0每次请求都要创建连接的缺点。http1.0需要使用参数keep-alive参数来告知服务器要建立一个长连接。
        3. host域： http1.0认为每台服务器都绑定了一个唯一的ip地址，因此请求消息的url 并不会传递主机名(hostname)，http1.0没有host域。随着虚拟主机技术的发展，在每一台物理服务器上可以存在多个虚拟主机( Multi-homed web servers )，并且它们共享一个ip地址。http1.1的请求消息和相应消息都支持host域，如果没有host域会报告错误 400 Bad Request
      • http1.1和http2.0的区别
        1. 多路复用： 一个TCP连接进行多个请求，http1.1虽然也可以但是多个请求之间的响应会被阻塞；
        2. http2.0采用二进制格式传输数据 在1.1版本中采用文本格式进行数据拆传输，二进制在协议的解析和优化拓展上有更多的优势；
        3. http2.0对消息采用HPACK进行压缩传输 在1.1版本中每次请求都会携带大量冗余头信息，浪费了很多宽带资源；
        4. 服务端推送： 服务端能够更快的把资源推送给客户端，而不需要客户端解析HTML再发送请求，当客户端需要的时候，它已经在客户端了
